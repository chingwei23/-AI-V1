▍第一部分：黃金五指框架

與 AI 溝通的核心心法，寫出精準 Prompt 的必備五元素。

💡 景崴講師的進階筆記

  • 1. 限制是靈魂不要只告訴 AI 要做什麼，一定要告訴它「絕對不能做什麼」。
  • 2. 保持批判性AI 是基於「已知」進行「未知預測」。請勿盲目相信，你是最終的決策者。
  • 3. 自我審查迴圈拿到答案後，再餵給 AI 問：「請以專家角度自評，這是最佳方案嗎？請提出優化建議。」
  • 4. 逆向工程思維企劃寫完後，務必補上一句：「請給我 5 個這個計畫可能會失敗的原因，並提供預防對策。」

▍第五部分：將系統部署上線

🌐 1. GitHub 免費網頁部署

  1. 前往 GitHub 官網 註冊並登入帳號。
  2. 點擊 New Repository 建立新專案。
  3. ⚠️ 極度重要：設定權限時，務必選擇「Public (公開)」，否則無法啟用網頁服務！
  4. 上傳剛剛寫好的 index.html 檔案並 Commit 存檔。
  5. 前往專案的 Settings > Pages，將 Source 選擇為 `main` 分支並儲存。等候幾分鐘，網址就會產出囉！
🔌

🔑 2. API 觀念與串接 (為網頁注入靈魂)

什麼是 API？

想像你去餐廳，API 就是「服務生」。你點餐（發送需求），服務生走到廚房（AI 伺服器）把做好的菜（文案結果）端回你的桌上（你的網頁）。

什麼是 API Key？

這是專屬的「信用卡通行證」。服務生要確認你有付費權限，才會幫你把需求送進頂級廚房。有了 Key，你的靜態網頁就擁有了 AI 算力！

👉 點我前往申請 Groq 免費 API Key

🖼️ 3. 網站放圖片的兩種法門

方法 怎麼做？ 優缺點比較
📂 方法一：同資料夾 (相對路徑) 把圖片上傳到 GitHub，和 index.html 放在同一個專案資料夾裡。程式碼寫 src="./logo.png" 最穩定、不會因為別人刪圖而破圖。
缺點：要手動上傳檔案。
🔗 方法二：網址連結 (絕對路徑) 把圖片上傳到圖床(如 Imgur)，直接在程式碼貼上網址：src="https://..." 程式碼乾淨、更換圖片只需換網址。
缺點：圖床掛掉或被刪除，網站就會破圖。
